# База знаний проекта: Trade Bot Hardcore Spec

## Инфраструктура и инструменты (MCP)
В этом проекте настроены следующие MCP-серверы (см. `mcp.json` для деталей):
- **filesystem** (npx): Доступ к файлам в `/home/yevge/trade-bot-hardcore-spec`.
- **ripgrep** (npx): Поиск по коду через `mcp-ripgrep`.
- **fetch** (uvx): Веб-доступ через `mcp-server-fetch`.
- **context7** (npx): Документация через `@upstash/context7-mcp`.
- **sonarqube** (docker): Анализ кода (требует запущенный Docker-демон).
- **mem0** (uvx): Долговременная память через `mem0-mcp-server`.
- **memory** (npx): Локальный граф знаний.
- **sequential-thinking** (npx): Пошаговое решение задач.

### Технические требования
Для работы всех инструментов в системе должны быть установлены:
- **Node.js / npm**: для запуска серверов через `npx`.
- **uv / uvx**: для высокоскоростного запуска Python-основанных MCP (например, `mem0` и `fetch`).
- **Docker**: обязателен для работы `sonarqube`.
- **Python 3**: необходим для работы некоторых вспомогательных скриптов.

## Полный список экспертных скиллов (.agents/skills/)
Каждый скилл доступен через команду `/skill:название`.

### Проектирование и планирование
- **design-an-interface**: Генерация нескольких вариантов дизайна интерфейса модуля.
- **request-refactor-plan**: Создание детального плана рефакторинга через интервью.
- **to-prd**: Превращение текущего контекста обсуждения в PRD (GitHub Issue).
- **to-issues**: Разбиение плана или PRD на независимые задачи (GitHub Issues).
- **grill-me**: Жесткое интервью по вашему плану для выявления слабых мест.
- **grill-with-docs**: Проверка плана на соответствие доменной модели и терминологии.

### Разработка и качество
- **tdd**: Цикл разработки через тестирование (Red-Green-Refactor).
- **diagnose**: Дисциплинированный цикл отладки и исправления сложных багов.
- **triage**: Исследование багов, поиск первопричины и создание плана исправления.
- **improve-codebase-architecture**: Поиск возможностей для углубления архитектуры и улучшения тестируемости.
- **migrate-to-shoehorn**: Миграция тестов с утверждений типа `as` на `@total-typescript/shoehorn`.
- **prototype**: Создание быстрых прототипов для проверки идей и дизайна.
- **qa**: Набор проверок качества для текущих изменений.

### Настройка окружения
- **setup-matt-pocock-skills**: Базовая настройка конфигов репозитория для работы других скиллов.
- **setup-pre-commit**: Настройка Husky, lint-staged, Prettier и тестов на пре-коммит.
- **git-guardrails-claude-code**: Установка хуков для блокировки опасных команд git (push, reset --hard).
- **scaffold-exercises**: Создание структуры директорий для учебных упражнений.

### Работа с текстом и знаниями
- **write-a-skill**: Создание новых структурированных скиллов.
- **edit-article**: Редактирование и улучшение текстов и статей.
- **ubiquitous-language**: Извлечение словаря терминов (Glossary) из обсуждения.
- **obsidian-vault**: Управление заметками в Obsidian (поиск, создание, вики-ссылки).
- **writing-beats / writing-fragments / writing-shape**: Инструменты для структурирования и улучшения качества текстов.

### Эффективность и коммуникация
- **caveman**: Ультра-короткий режим общения для экономии ~75% токенов.
- **zoom-out**: Запрос высокоуровневого обзора или контекста по незнакомому коду.
- **handoff**: Создание документа передачи дел для другого агента.

## Стандарты разработки
- **Никаких галлюцинаций**: Не выдумывай API. Если не уверен — используй `context7` или `fetch` для проверки документации.
- **Работа с памятью**: Важные решения сохраняй в `mem0`. Общие правила проекта — в этот файл (`knowledge.md`).
- **TDD прежде всего**: Пиши тесты перед реализацией кода, используя скилл `tdd`.
- **Язык**: Основной язык общения и инструкций — русский.
