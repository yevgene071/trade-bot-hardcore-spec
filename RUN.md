# Инструкция по запуску

После успешной сборки проекта (см. `BUILD.md`), используйте эти команды для запуска.

## 1. Запуск торгового робота

Основной исполняемый файл находится в `build/release/bin/trade_bot`.

```bash
# Запуск с конфигурационным файлом по умолчанию
./build/release/bin/trade_bot --config config.toml

# Запуск в фоновом режиме (через nohup)
nohup ./build/release/bin/trade_bot --config config.toml > logs/bot.log 2>&1 &
```

**Основные параметры командной строки:**
- `--config <path>`: путь к файлу конфигурации (TOML).
- `--log-level <level>`: уровень логирования (trace, debug, info, warn, error).

---

## 2. Запуск тестов

Для проверки корректности работы всех компонентов используйте `ctest`.

```bash
# Запуск всех тестов из директории сборки
ctest --test-dir build/release --output-on-failure

# Запуск конкретного теста (например, только стратегии)
./build/release/bin/bounce_strategy_test
```

---

## 3. Мониторинг и логи

- **Логи в терминале:** По умолчанию выводятся в `stdout`.
- **Файлы логов:** Если настроено в `config.toml`, логи будут сохраняться в папку `logs/`.
- **Dashboard:** Если `DashboardServer` включен, вы можете открыть веб-интерфейс (адрес и порт указываются в конфиге, обычно `http://localhost:8080`).

---

## 4. Полезные команды

```bash
# Проверка версии и доступных опций
./build/release/bin/trade_bot --help

# Просмотр логов в реальном времени
tail -f logs/trade_bot.log
```
