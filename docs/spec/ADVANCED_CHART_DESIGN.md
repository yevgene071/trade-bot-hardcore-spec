# Advanced Chart Visualization — Design Specification

> **Статус:** Готово к реализации · В разработке  
> **Дата:** 2026-05-16  
> **Scope:** Переосмысление визуализации рынка — отказ от классических свечей в пользу микроструктурного анализа

---

## 0. Анализ текущего состояния

### Что уже реализовано (не нужно делать)

| Компонент | Статус | Где |
|-----------|--------|-----|
| `ChartPoint` C++ структура со всеми полями | ✅ Готово | `core/src/features/ChartHistory.hpp` |
| `ChartPoint` TypeScript тип со всеми полями | ✅ Готово | `dashboard/src/types.ts` |
| Передача `chart_history[]` через WebSocket | ✅ Готово | `DashboardServer` → `useTradeStore` |
| `orderbook.bids` / `orderbook.asks` в store | ✅ Готово | `useTradeStore.orderbook` (top20) |
| Все поля `tape_aggression`, `leader_*`, `imbalance`, `prints_per_sec` | ✅ Готово | В `ChartPoint` и store |

**Фаза 1 из плана внедрения полностью выполнена.**

### Что есть сейчас

Текущий график (`Chart.tsx`) использует библиотеку **lightweight-charts** с тремя стандартными сериями:

| Слой | Тип | Данные |
|------|-----|--------|
| Area Series | Ценовой тренд (mid, зелёный градиент) | `chartHistory[].mid` |
| Candlestick Series | OHLC свечи (1-мин агрегация) | `chartHistory[].mid` → синтетические свечи |
| Histogram Series | Объём (синий, снизу) | `buy_vol_5s + sell_vol_5s` |
| Markers | Сигналы (стрелки buy/sell) | `signals[]` |

**Проблема:** Это классический подход, который игнорирует 80% данных, доступных в `FeatureFrame`:

- `tape_aggression` — агрессия ленты (buy vs sell pressure)
- `leader_correlation`, `leader_lag_ms` — лидер-ведомый динамика
- `imbalance` — дисбаланс стакана
- `volatility_1min_bps` — микро-волатильность
- `prints_per_sec` — скорость потока сделок
- `nearest_support`, `nearest_resistance` — структурные уровни
- `bid_depth_10`, `ask_depth_10` — глубина стакана

Классические свечи **скрывают микроструктуру** рынка. Трейдер видит только OHLC, но не видит **почему** цена движется: агрессивно или пассивно, с поддержкой лидера или вопреки, в зоне плотности или в пустоте.

## 1. Философия нового подхода

### Принципы

| # | Принцип | Описание |
|---|---------|----------|
| **P1** | **Многомерность** | Цена × время — лишь проекция. Показываем скрытые измерения: агрессию, плотность, корреляцию, скорость |
| **P2** | **Причинность** | Визуализация должна отвечать не «что случилось», а «почему»: почему бот вошёл, почему отскочили, почему пробили |
| **P3** | **Топология ликвидности** | Ликвидность — не линия, а ландшафт. Показываем её как непрерывное поле с пиками, провалами и стенами |
| **P4** | **Режимы рынка** | Рынок переключается между режимами (тренд, реверсия, флэш). Визуализация должна делать это очевидным |
| **P5** | **Поток, а не статика** | Данные непрерывны. Применяем идеи из визуализации потоков (flow visualization), а не дискретных баров |
| **P6** | **Интуитивная сложность** | Сложные данные → интуитивно понятная картинка. Цвет, толщина, текстура, движение — всё несёт смысл |

### От чего отказываемся

- ❌ **OHLC свечи** — агрегируют и скрывают микроструктуру
- ❌ **Дискретные volume-бары** — не показывают природу объёма (агрессивный vs пассивный)
- ❌ **Равномерная сетка времени** — рынок не течёт равномерно; событийное время информативнее
- ❌ **Единая цена (mid)** — bid и ask живут своей жизнью; их расхождение информативно

## 2. Концепции визуализации

### 2.1 Топологическая карта ликвидности (Liquidity Topology Map) ★ PRIMARY

**Идея:** Заменить фон графика на непрерывное поле ликвидности. Цена движется как тропа через ландшафт, где «горы» — зоны концентрации ликвидности (стены ордеров), а «долины» — разрежённые зоны.

#### Как выглядит

```
Цена ↑
     │
67260 ┤     ╱╲░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
     │    ╱  ╲░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
67240 ┤───╱────╲────████████──╱╲───────────────────────────────████──
     │  ╱      ╲   ██ WALL ██ ╱  ╲                              ██
67220 ┤─╱────────╲──████████─╱────╲────██████────────────╱╲────████──
     │╱          ╲           ╲    ╲───██████───────────╱  ╲
67200 ┤───────────╲───────────╲──────────────╱╲───────╱────╲─────────
     │            ╲           ╲             ╱  ╲     ╱      ╲
67180 ┤───────────╲───────────╲───────────╱────╲───╱────────╲───────
     │            ╲           ╲          ╱      ╲─╱          ╲
67160 ┤────────────╲───────────╲────────╱────────╲────────────╲─────
     │             ╲           ╲                       
67140 ┤──────────────╲───────────╲──────────────────────────────────
     │               ╲           ╲
     └────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────→ Время
        08:00    10:00    12:00    14:00    16:00    18:00    20:00

Условные обозначения:
  ░░░ = низкая плотность (пустота, легко пробить)
  ███ = высокая плотность (стена, отскок вероятен)
  ─── = цена (mid) — тропа через ландшафт
```

#### Ключевые визуальные элементы

| Элемент | Данные | Визуализация |
|---------|--------|--------------|
| **Поле плотности** | `bids_top20`, `asks_top20`, `bid_depth_10`, `ask_depth_10` | Непрерывный тепловой фон: холодный (тёмный) = пусто, горячий (янтарный) = плотность. Интерполируется между уровнями стакана |
| **Ценовая линия** | `mid` | Белая/светлая линия. Толщина меняется: толще при высокой `tape_aggression` |
| **Стены (Density Walls)** | Top20 уровней с size > порога | Янтарные/золотые полосы на ценовых уровнях. Пульсируют при приближении цены |
| **Bid/Ask расхождение** | `best_bid`, `best_ask` | Тонкая тень вокруг линии цены = область спреда. Расширяется при росте `spread_bps` |
| **Айсберг-маркеры** | IcebergDetector сигналы | Радиальная пульсация (как сонар) от ценового уровня, где обнаружен скрытый ордер. Затухает за 3-5 секунд. **⚠️ Отложено в Фазу 2** — данные IcebergDetector не передаются на фронт через WS (нет поля в state). |

#### Почему это работает для бота

- **DensityDetector** → прямо видно зоны плотности как горячие пятна
- **BounceFromDensity** → цена приближается к горячей зоне и отскакивает: визуально очевидно
- **BreakoutEatThrough** → цена «проедает» горячую зону (плотность тает на глазах)
- **IcebergDetector** → сонарные метки показывают, где бот видит скрытую ликвидность

#### Реализация

- **Технология:** HTML5 Canvas 2D или WebGL (для плавного рендеринга поля)
- **Данные на вход:** `chartHistory[]` (mid) + `bids_top20[]` + `asks_top20[]` (обновляются с бэкенда)
- **Интерполяция:** Поле плотности интерполируется бикубически между известными уровнями
- **Производительность:** ~300 точек истории, поле 540×400 px → легко укладывается в 16ms на Canvas

### 2.2 Лента агрессии: Flow Ribbon (Tape Aggression Flow) ★ SECONDARY

**Идея:** Заменить volume-бары на непрерывную «ленту потока», которая показывает не просто объём, а природу каждой сделки: агрессивный покупатель (market buy) или агрессивный продавец (market sell).

#### Как выглядит

```
         Buy pressure (aggressive market buys)
         ┌──────────────────────────────────────┐
         │██████████████████░░░░░░░░░░░░░░░░░░░░│ ← Зелёный = buy aggression
    mid ─────────────────────────────────────── → Цена
         │░░░░░░░░░░░░░░░░░░░░██████████████████│ ← Красный = sell aggression
         └──────────────────────────────────────┘
         Sell pressure (aggressive market sells)
         
Ширина ленты = prints_per_sec (количество сделок в секунду)
Цвет = tape_aggression (−1 = чистый sell, 0 = пассивный, +1 = чистый buy)
Насыщенность = величина дисбаланса покупок/продаж
```

#### Четыре паттерна, которые видно мгновенно

| Паттерн | Визуализация | Значение | Стратегия |
|---------|-------------|----------|-----------|
| **Зелёный импульс** | Лента резко зеленеет и расширяется | Агрессивные покупки, цена растёт | LeaderLag long |
| **Красный импульс** | Лента краснеет и расширяется | Агрессивные продажи, цена падает | LeaderLag short |
| **Flush Reversal** | Резкий красный всплеск → мгновенный разворот в зелёный | Вымывание продавцов, разворот | **FlushReversal** |
| **Затухание** | Лента сужается, цвет бледнеет | Пассивный рынок, низкая активность | Бот не входит, ждёт |

#### Техническая реализация

- **Позиция:** Нижняя панель под основным графиком (25% высоты)
- **Данные:** `tape_aggression`, `prints_per_sec`, `buy_vol_5s`, `sell_vol_5s`
- **Рендер:** Разделённая область: верхняя половина — buy pressure, нижняя — sell pressure. Каждый тик добавляет сегмент ленты. Цвет определяется знаком и величиной `tape_aggression`
- **Сглаживание:** Экспоненциальное сглаживание с периодом ~200ms для плавности

### 2.3 Лидер-Ведомый: «Призрачная» траектория (Leader-Lag Ghost Field) ★ TERTIARY

**Идея:** Наложить на график «призрачную» линию — куда бы пошла цена, если бы она идеально следовала за лидером. Расстояние между реальной ценой и призраком показывает силу корреляции. Зазор между ними — зона «свободы» от лидера.

#### Как выглядит

```
Цена ↑
     │              ╱╲ реальная цена (белая)
67220 ┤      ╱──────╱──╲────╲
     │     ╱      ╱    ╲    ╲────╲
67200 ┤────╱─────╱──────╲─────────╲────
     │   ╱     ╱        ╲         ╲
67180 ┤──╱────╱──────────╲─────────╲──
     │ ╱    ·············╲·········╲··
67160 ┤╱    ·  призрак  ··╲·········╲·
     │     · (лидер +    · ╲        ·╲
67140 ┤────·──lag_ms)────·──╲───────·─╲
     │   ·               ·   ╲     ·
     └───·───┬────┬────┬─·──┬─╲───┬─·──→
         · 08:00    10:00  · 12:00 ·
         
Условные обозначения:
  ─── = реальная цена
  ··· = «призрак» лидера (leader price, сдвинутый на leader_lag_ms)
  Зелёное свечение между линиями = высокая корреляция (>0.8)
  Красноватый зазор = расхождение (корреляция упала)
  Конус = зона предсказанного движения
```

#### Что это показывает

| Ситуация | Визуальный сигнал | Действие бота |
|----------|-------------------|---------------|
| Линии слипаются | Высокая корреляция, зелёное свечение | **LeaderLag** активно торгует |
| Призрак уходит вверх, цена отстаёт | Красноватый зазор расширяется | Бот ждёт догоняющего движения |
| Призрак разворачивается | Лидер меняет направление | Сигнал для всего universe |
| Внезапный разрыв | Резкое падение корреляции | Десинхронизация, бот выходит из позиций |

#### Данные

- `leader_change_1s` — дробное изменение цены лидера за 1 сек (единицы: ≈ ±0.0001..0.001)
- `leader_correlation` — определяет цвет зазора (зелёный→жёлтый→красный)
- `leader_lag_ms` — задержка, с которой ведомый следует за лидером

#### Уточнённая формула призрака

```
TICK_MS ≈ 100  // cadence chart_history (100ms)

lagTicks(i) = max(0, round(history[i].leader_lag_ms / TICK_MS))
basePt(i)   = history[max(0, i - lagTicks(i))]
ghostMid(i) = basePt(i).mid × (1 + history[i].leader_change_1s × history[i].leader_correlation)
```

Если `ghost_mid` визуально слишком далеко от реальной цены — добавить damping: умножить `leader_change_1s` на 0.1.  
Призрак рисуется только при `leader_correlation > 0.3`.

### 2.4 Фазовый портрет: Карта рыночного режима (Microstructure Phase Space) ★ QUATERNARY

**Идея:** Отдельная панель (не на основном графике), показывающая текущее состояние рынка в абстрактном фазовом пространстве. Это не график цены, а график **состояния рынка**.

#### Как выглядит

```
           Volatility ↑ (1min bps)
                      │
        15 ┤          │   ●← текущая точка
           │    ФЛЭШ  │  ╱  (vol=12, imb=+0.6)
        12 ┤   (Flush │ ╱   режим: MOMENTUM
           │   Reversal)╱   стратегия: BreakoutEatThrough
         9 ┤          ╱
           │         ╱
         6 ┤        ╱
           │    ●──╱────●
         3 ┤   ╱  НАКОПЛЕНИЕ
           │  ╱   (Bounce)
         0 ┼──┼────┼────┼──→ Imbalance
          -1   -.5  0   +.5   +1
              sell          buy
              
Условные обозначения:
  ● = исторические точки (след за последние 120 тиков), затухают
  ● = текущая точка, яркая, пульсирует
  Зоны:  MOMENTUM, MEAN-REVERSION, ACCUMULATION, DISTRIBUTION, FLUSH
```

#### Четыре квадранта → четыре режима

| Квадрант | Режим | Volatility | Imbalance | Активная стратегия |
|----------|-------|-----------|-----------|-------------------|
| **I** (↗→) | **MOMENTUM** | Высокая | Buy pressure | **BreakoutEatThrough**, LeaderLag long |
| **II** (↖→) | **FLUSH/REVERSAL** | Высокая | Sell pressure | **FlushReversal** |
| **III** (↙←) | **DISTRIBUTION** | Низкая | Sell pressure | BounceFromDensity short |
| **IV** (↘←) | **ACCUMULATION** | Низкая | Buy pressure | **BounceFromDensity** long |
| Центр | **NEUTRAL** | Низкая | ~0 | Ожидание |

#### Почему это мощно

- Заменяет интуицию «рынок выглядит вялым» на объективную карту
- Показывает переходы между режимами: точка медленно дрейфует из ACCUMULATION в MOMENTUM
- Объясняет, **почему** бот выбрал стратегию X, а не Y — потому что рынок в соответствующем режиме
- Исключает субъективность: вместо «мне кажется» → «точка в правом верхнем квадранте»

#### Реализация

- **Позиция:** Мини-панель 200×200 px в правой части дашборда (рядом с сигналами)
- **Данные:** `volatility_1min_bps`, `imbalance`, `tape_aggression`
- **Отрисовка:** Canvas 2D scatter plot. След из последних 120 тиков. Координаты следа: X=`imbalance`, Y=`volatility_1min_bps`. Размер точки = `prints_per_sec`
- **Анимация:** Точка движется плавно (интерполяция между тиками)

## 4. Технический дизайн

### 4.1 Технологический стек

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| **Топологическая карта** | HTML5 Canvas 2D + OffscreenCanvas | Достаточно для 300 точек. WebGL избыточен на данном этапе |
| **Лента агрессии** | Canvas 2D | Простая геометрия, 2 цветовых канала |
| **Фазовый портрет** | Canvas 2D | Scatter plot, 120 точек |
| **Призрак лидера** | Canvas 2D (отдельный слой) | Наложение на основной график |
| **Анимация** | `requestAnimationFrame` | Стандарт для 60fps |

### 4.2 Поток данных

```

chartHistory[].tape_aggression   → Лента агрессии (цвет, ±1)
chartHistory[].buy_vol_5s        → Лента агрессии (высота buy)
chartHistory[].sell_vol_5s       → Лента агрессии (высота sell)
chartHistory[].spread_bps        → Спред-тень вокруг ценовой линии
chartHistory[].leader_correlation→ Призрак лидера (цвет зазора)
chartHistory[].leader_change_1s  → Призрак лидера (дельта цены)
chartHistory[].leader_lag_ms     → Призрак лидера (сдвиг)
chartHistory[].volatility_1min_bps→ Фазовый портрет (Y)
chartHistory[].imbalance         → Фазовый портрет (X из истории)
chartHistory[].prints_per_sec    → Размер точки в фазовом портрете
```

> **Важно:** `orderbook.bids/asks` — это **только текущий снимок** стакана, без истории.
> Поле плотности отображает актуальное состояние ликвидности, а не ретроспективу.
> Для истории плотности потребуется отдельный буфер на бэкенде (Фаза 2).

## 6. Сравнение с классикой

| Аспект | Классические свечи | Предлагаемый подход |
|--------|-------------------|---------------------|
| **Измерения** | 2D: цена × время | 5D+: цена, время, плотность, агрессия, корреляция, скорость |

| **Режим рынка** | Трейдер гадает | Фазовый портрет — объективная карта |
| **Для кого** | Ритейл-трейдеры | Алго-трейдеры и кванты |

## 8. Альтернативные/дополнительные идеи

### 8.1 Спектрограмма микроструктуры

Вертикальная полоса справа от графика, показывающая частотный спектр движения цены (через FFT на лету). Трендовый рынок = низкие частоты доминируют. Чоппи = высокие частоты. Позволяет мгновенно отличить тренд от боковика.

### 8.2 Событийное время (Event Time)

Вместо равномерной шкалы времени — шкала, где каждый тик = одна единица. Рынок «ускоряется» при высокой активности. Это показывает реальную плотность событий.

### 8.3 Bid/Ask расхождение как tension

Линии bid и ask рисуются отдельно, расхождение между ними показано цветовым градиентом (как натяжение резины). Когда спред узкий — линии близко, градиент спокойный. Когда спред расширяется — натяжение растёт.

---

## 9. Резюме

Предлагаемый подход заменяет классическую парадигму «цена + объём» на многомерную визуализацию **микроструктуры рынка**:

1. **Топологическая карта ликвидности** — цена как тропа через ландшафт плотности
2. **Лента агрессии** — поток сделок с различением агрессивных и пассивных
3. **Призрак лидера** — гравитационное поле лидер-ведомый
4. **Фазовый портрет** — объективная карта рыночного режима

Эти четыре компонента вместе дают полную картину: **не только «что», но и «почему»**. Это именно то, что нужно алго-трейдеру для понимания поведения бота и рынка.

## Приложение C: Алгоритм поля плотности

```
Входные данные: bids_top20[], asks_top20[], priceMin, priceMax, canvasH

sigma = (priceMax - priceMin) * 0.005
allLevels = [...bids, ...asks]  // до 40 уровней

// Шаг 1: вычислить density для каждой строки пикселей
for y in 0..canvasH:
  rowPrice = priceMin + (1 - y/canvasH) * (priceMax - priceMin)
  density[y] = Σ level.size × exp(-0.5 × ((rowPrice - level.price) / sigma)²)

// Шаг 2: нормализация
maxDensity = max(density[])

// Шаг 3: colormap (для каждой строки → заливка всей ширины одним цветом)
t = sqrt(density[y] / maxDensity)  // sqrt для контраста на краях
r = lerp(4,  192, t)
g = lerp(6,  136, t)  
b = lerp(12,  40, t)
a = t * 200  // alpha 0..200

// Шаг 4: wall lines — горизонтальные линии для уровней выше порога
wallThreshold = percentile80(allLevels.map(l => l.size))
for level in allLevels where level.size >= wallThreshold:
  drawHorizontalLine(y=priceToPixel(level.price), color=wallAmber)
```

Оптимизация: использовать `OffscreenCanvas` и `createImageData` для pixel-by-pixel рендера.
