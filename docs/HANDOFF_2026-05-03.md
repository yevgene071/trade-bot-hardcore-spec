## Windows pickup checklist

### 1. Build prerequisites

You'll need:

- Visual Studio 2022 17.7+ (C++23 support: `/std:c++latest`) OR clang-cl OR MinGW-w64 with gcc-13+
- CMake ≥ 3.22
- Ninja
- Python 3.10+ + pipx + Conan 2.x
- Git

```powershell
# elevated PowerShell, one-shot:
winget install -e --id Kitware.CMake
winget install -e --id Ninja-build.Ninja
winget install -e --id Python.Python.3.12
python -m pip install --user pipx
python -m pipx ensurepath
pipx install conan
```

**Conan profile** — this part is fragile across OSes. Do `conan profile detect --force` then verify:

```ini
[settings]
arch=x86_64
compiler=msvc
compiler.cppstd=17        # not 23 — keeps prebuilt binary cache hits
compiler.runtime=dynamic
compiler.version=193
build_type=Release
os=Windows
```

C++23 is set per-target in `CMakeLists.txt`, not via Conan profile, so cppstd=17 in the profile is fine and avoids re-building boost/abseil from source (those would take 30+ min).

### 2. Known issues to expect on Windows

- **OpenSSL Conan recipe bug** (already fixed in our `CMakeLists.txt`): `INTERFACE_INCLUDE_DIRECTORIES` is empty for `openssl/3.6.2`. The fallback that force-attaches the path from `openssl_PACKAGE_FOLDER_RELEASE` is in place — should work cross-platform.
- **inotify** in `NewsCalendar.cpp` is Linux-only. The hot-reload watcher won't compile on Windows. Either:
  - Stub the watcher with a polling fallback (file mtime check every 1 s), OR
  - Wrap the inotify code in `#ifdef __linux__` and skip watch on Windows (SIGHUP also doesn't exist there — the manual `reload()` API still works).
  This is the **first task** on Windows.
- **POSIX `flock`** in `AccountStatePersister.cpp` is Linux. Use `LockFileEx` on Windows or wrap in `#ifdef __linux__` — losing the lock just means losing the cross-process safety net, not correctness.
- **`signalfd`-style patterns** (we don't use any directly, but watch out): SIGINT/SIGTERM are still supported via `signal()` on Windows.
- **`pthread_setaffinity_np`** in `CpuPinning.cpp` doesn't exist. Use `SetThreadAffinityMask` from WinAPI or stub for Windows.
- **`getrusage`** in tests: replace with `GetProcessMemoryInfo`.
